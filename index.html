Towards Accessible Survival Analysis: A Reproducible Framework with Integrated Interpretability and GUI-Based Interaction

Paolo Sorino, Domenico Lofù, Martino Trivisano, Leonardo Capano, Tommaso Di Noia, Eugenio Di Sciascio

Department of Electrical and Information Engineering, Polytechnic of Bari, 70125 Bari, Italy

Watch Interactive Demo

Abstract

Reproducibility, deployment consistency, and usability remain critical challenges in survival analysis, particularly when integrating heterogeneous modeling approaches into real-world workflows. This work presents a user-oriented and reproducible framework for survival analysis that unifies statistical, machine learning (ML), and deep learning (DL) models within a consistent and accessible analytical environment. The framework enforces standardized preprocessing, deterministic data partitioning, and survival-aware validation, while an artifact-based design ensures consistent reconstruction of the analytical workflow during inference. A graphical user interface (GUI) with role-based access control enables both technical and non-technical users to interact with the system through structured and guided workflows. Interpretability is integrated through feature attribution methods based on SHapley Additive exPlanations (SHAP), complemented by controlled natural-language explanations.

The Analytical Ecosystem

A modular architecture designed for advanced and robust mortality modeling.

1. The Problem and the Data

Survival analysis methods generally require strong programming skills, which limits the adoption of advanced models by domain experts (e.g., healthcare professionals). The framework was tested on a real-world administrative cohort from the municipality of Taranto (a reproducible subsample of 5,000 observations). The dataset includes demographics and long-term environmental exposure variables (NO₂, PM₁₀, PM₂.₅).

2. Preprocessing and Pipeline

The system guarantees zero "data leakage" by enforcing strict deterministic transformations:

  • Removal of features with a missing value rate > 50%.
  • One-Hot Encoding for categorical variables.
  • Transformations are fitted exclusively on the training set and identically applied to the test set to preserve evaluation integrity.

3. Heterogeneous Models Supported

The modeling layer integrates classical and Artificial Intelligence approaches through a unified computational interface, including:

  • Statistical Models: Kaplan-Meier, Cox Proportional Hazards.
  • Machine Learning: Random Survival Forest (RSF), Gradient Boosting (GB), Survival SVM.
  • Deep Learning: DeepSurv (optimized via Optuna).

4. Explainability (XAI)

Interpretability modules are embedded as core components of the analytical pipeline:

  • SHAP: For local and global feature importance attribution.
  • DiCE: To generate counterfactual scenarios.
  • LLM: A Language Model generates natural-language explanations strictly anchored to the analytical outputs, effectively preventing hallucinations.

Experimental Results

The experimental results demonstrate that when heterogeneous survival models are executed within a shared and strictly controlled pipeline, they achieve comparable predictive performance. This highlights the critical importance of reproducibility and consistent configuration over model-specific differences.
